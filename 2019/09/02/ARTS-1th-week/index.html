Algorithm

题目No104：给定一个二叉树，找出其最大深度。

二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

说明: 叶子节点是指没有子节点的节点。

示例：给定二叉树 [3,9,20,null,null,15,7]，

​ 3

/ \
9 20
/ \
15 7
返回它的最大深度 3 。

解题思路：像这种问题首先考虑递归实现。由于是普通二叉树，存在各种极端情况，需要遍历到每一个叶子结点。叶子节点为null的，则为此条线路最深的节点，递归一次深度需要加一，直到子节点为null。

JSR-160、JMX规范

https://download.oracle.com/otn-pub/jcp/jmx_remote-1_4-mrel2-eval-spec/jsr160-jmx-1_4-mrel4-spec-FINAL-v1_0.pdf

​ 开始接触JMX规范是在学习Jolokia的时候，jolokia实现了JMX规范，并且能监控应用，感觉JMX特别强大。今天了解了一下1.1-1.3章节的介绍，感觉真的国外的计算机概念很超前。这个规范是2013年发布的，那个时候JMX作为分布式应用的萌芽期，JMX规范已经布局分布式系统了，JMX规范是能够为分布式系统服务的。

​ 本文的1.1-1.3章节主要介绍了，JMX架构的概念，使用JMX规范的好处，此处需要强调的是，之前一直认为JMX规范注重的是应用监控，但是读了原文才发现，其实JMX本身更强调的是JMX规范的management and monitoring，这两方面的功能，之前真是一叶蔽目，认识粗浅。JMX的益处总结在于：低成本，可扩展，易集成。实在是为引入JMX规范操碎了心。其中还包括：compatibility test suite(CTS)兼容测试组件，方便测试。JMX规范的实现范围主要是三方面的，设备层面，代理层面，分布式层面。不是特别理解，下个星期再好好看看吧。

这篇文章是微信公众号EAWorld发表的，这个公众号是一个难得的质量高，但是不火的公众号。

本篇文章主要介绍了保证应用安全的五个方面：身份认证，访问授权，通信安全，代码安全，管理审计。传统概念一般强调前两点，认证与授权，但是后面三点依然是比较重要的，B站的源码泄露就是管理审计没做好，还好B站的代码做好了，不至于让别人从源码中获取非法资源。

其实本片文章还是在身份认证上描述的比较详细，主要是推荐微服务架构使用OAuth2.0 授权协议来搭建IAM系统。Spring 体系可以基于Spring Security OAuth实现授权服务器和客户端。

主要介绍了在OAuth2.0授权协议中，主要定义了四种许可类型：授权码许可、简单许可、密码凭据许可和客户端凭据许可。这里面的参考意义比较大。
